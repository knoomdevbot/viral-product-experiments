# UI Guidelines: Viral Product Experiments

Last updated: 2026-06-29
Owner: NED UI Designer

## Product context
Two mobile-first static MVPs for social entertainment validation. Each app should feel like a native consumer artifact, not a developer demo.

## Shared rules
- Mobile-first: primary journey fits a 390–430px viewport with one obvious next action per section.
- Result artifact first: generated card/poster must be visually dominant and screenshot/download-worthy.
- Actions after result: regenerate, share URL/caption, and download must be visible without hunting.
- Validation: inline error copy for blocked steps; avoid silent no-ops.
- Accessibility: semantic headings, labeled controls, 44px-ish touch targets, visible focus states, sufficient contrast.
- Static MVP transparency: note that it runs locally with no account/API and provide feedback link.

## VibeCard visual direction
- Benchmarks: Korean personality/result cards, idol photo cards, fortune cards, collectible profile artifacts.
- Light card-studio surface; cream/lavender/pink/mint palette.
- The generated card should include serial/detail labels, title, verdict, stats, charm, and watermark.
- Copy tone: playful, concise, collectible.

## ShowDrop visual direction
- Benchmarks: episodic interactive storytelling, streaming poster packaging, role-claim social loops.
- Dark cinematic surface; neon red/violet/cyan/gold palette.
- Generated show package must include poster, episode beat, cast cards, relationship board, role claim, and Episode 2 unlock.
- Copy tone: cinematic, social, group-lore focused; do not frame as companion chat.
